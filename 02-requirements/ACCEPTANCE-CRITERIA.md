# ACCEPTANCE-CRITERIA.md - 验收标准

基于PRD生成的可测试验收标准。

---

## AC-2.1 智能解析与数据中枢

### AC-2.1.1 多格式文档解析

**Word文档解析**
- [ ] 当上传.docx文件时，系统应成功解析文档内容
- [ ] 当文档包含表格时，系统应正确提取表格数据
- [ ] 当文档包含页眉页脚时，系统应忽略这些内容

**PDF文档解析**
- [ ] 当上传文字型PDF时，系统应成功提取文本内容
- [ ] 当上传扫描型PDF时，系统应调用OCR进行识别
- [ ] 当PDF包含多页时，系统应完整解析所有页面

**准确率验证**
- [ ] 对100份测试合同，关键字段提取准确率应≥85%

### AC-2.1.2 合同类型识别

- [ ] 系统应优先根据合同标题识别类型
- [ ] 当标题无法识别时，应根据合同内容关键词识别
- [ ] 当上传人力框架合同时，系统应自动识别为"人力框架"类型
- [ ] 当上传项目外包合同时，系统应自动识别为"项目外包"类型
- [ ] 当上传产品购销合同时，系统应自动识别为"产品购销"类型
- [ ] 当无法自动识别时，系统应提示用户手动选择

### AC-2.1.3 关键字段提取

**通用核心字段（所有合同）**
- [ ] 系统应提取身份标识：合同编号、合同名称
- [ ] 系统应提取签约方信息：我方主体、客户名称、客户统一社会信用代码
- [ ] 系统应提取财务条款：合同总金额（含税/不含税）、币种、税率、付款方式、付款条件
- [ ] 系统应提取时间周期：签署日期、生效日期、终止日期、合同期限
- [ ] 系统应提取状态分类：合同类型、所属行业、业务部门/销售负责人
- [ ] 当字段无法提取时，系统应标记为"待补充"

**人力框架合同特定字段**
- [ ] 系统应提取人员级别/角色及对应费率
- [ ] 系统应提取预计总工时/人日、工时上限
- [ ] 系统应提取结算周期、工时调整机制

**项目外包合同特定字段**
- [ ] 系统应提取SOW范围摘要、关键交付物清单
- [ ] 系统应提取里程碑名称、交付物、金额/比例、计划完成日期
- [ ] 系统应提取验收标准、范围变更流程

**产品购销合同特定字段**
- [ ] 系统应提取产品/服务清单、规格配置、数量、单价
- [ ] 系统应提取交付内容、交付日期、交付地点
- [ ] 系统应提取保修期、售后服务条款

## AC-2.2 财务视图

### AC-2.2.1 收入仪表盘

- [ ] 当查看人力合同时，应显示基于工时的收入计算
- [ ] 当查看项目合同时，应显示按里程碑的收入确认进度
- [ ] 仪表盘数据应在合同更新后5分钟内刷新

### AC-2.2.2 现金流预测

- [ ] 系统应基于付款条款生成12个月滚动预测
- [ ] 应支持按客户、按项目筛选查看
- [ ] 应高亮显示未来30天待收款项

### AC-2.2.3 预警

- [ ] 当付款逾期超过7天时，应发送预警邮件
- [ ] 预警应包含合同编号、客户名称、逾期金额

---

## AC-2.3 交付视图

### AC-2.3.1 交付地图

- [ ] 应显示所有执行中合同的状态（红/黄/绿）
- [ ] 红灯：逾期或严重风险；黄灯：有风险；绿灯：正常
- [ ] 点击合同应跳转详情页

### AC-2.3.2 里程碑跟踪

- [ ] 应从合同中自动提取里程碑信息
- [ ] 里程碑到期前7天应发送提醒
- [ ] 应支持手动标记里程碑完成

---

## AC-2.4 业务/销售视图

### AC-2.4.1 客户360°

- [ ] 应聚合显示客户所有历史合同
- [ ] 应计算并显示客户LTV
- [ ] 应显示合同续约历史时间线

### AC-2.4.2 续约管理

- [ ] 合同到期前90天应发送首次提醒
- [ ] 到期前60天、30天应发送后续提醒
- [ ] 应显示客户健康度评分（0-100）

---

## AC-2.5 市场视图

### AC-2.5.1 智能检索

- [ ] 应支持关键词全文检索
- [ ] 应支持语义相似搜索
- [ ] 搜索结果应在2秒内返回

### AC-2.5.2 案例生成

- [ ] 应能基于合同生成Markdown格式的Case Study
- [ ] 生成内容应包含：客户挑战、解决方案、成果
- [ ] 应支持敏感信息脱敏选项

---

## AC-2.6 法务/风控视图

### AC-2.6.1 条款合规扫描

- [ ] 应自动与公司标准模板对比
- [ ] 应标记偏差条款
- [ ] 高风险条款应特别预警

### AC-2.6.2 风险评分

- [ ] 应为每份合同计算风险评分（0-100）
- [ ] 评分因素应包含：客户信用、付款历史、条款风险

---

## AC-2.7 管理层视图

### AC-2.7.1 健康度仪表盘

- [ ] 应显示核心指标：总签约额、在执合同数、DSO
- [ ] 应显示风险热力图
- [ ] 数据应实时更新

## 非功能验收标准

### AC-NF-1 性能

- [ ] 首页加载时间应<2秒
- [ ] 合同解析时间应<30秒（10页以内）
- [ ] 系统应支持10并发用户

### AC-NF-2 安全

- [ ] 所有API应通过HTTPS
- [ ] 用户密码应加密存储
- [ ] 应实现RBAC权限控制

### AC-NF-3 可用性

- [ ] 系统可用性应≥99.5%
- [ ] 数据应每日自动备份
